📋 Project Overview

The AI Insurance Advisory & Policy Recommendation Platform is a modern REST API-based system designed to:

  • Manage user profiles and insurance policies
  • Provide AI-driven policy recommendations
  • Handle insurance claims processing
  • Generate personalized insurance advisory
  • Track recommendation acceptance and claims status

⚙️ Technology Stack

Backend FastAPI
Database SQLite
ORM SQLAlchemy
Validation Pydantic
Server Uvicorn
Testing Postman

✨ Key Features

👥 User Management

Create and manage user profiles with demographic and health information for personalized recommendations.

Browse comprehensive insurance policies across multiple categories: Health, Life, Auto, and Property.

AI-powered recommendations with scoring system based on user profile and insurance needs.

🔌 API Endpoints

Health & Information

GET /health - Health check endpoint
GET / - Root endpoint with API information

User Management

GET /users - List all users (paginated)
GET /users/{user_id} - Get specific user
POST /users - Create new user
PUT /users/{user_id} - Update user information
DELETE /users/{user_id} - Delete user

Policy Management

GET /policies - List active policies
GET /policies/{policy_id} - Get specific policy
GET /policies/type/{policy_type} - Get policies by type
POST /policies - Create new policy

Recommendations

POST /recommendations - Create recommendation
GET /recommendations/user/{user_id} - Get user recommendations
PUT /recommendations/{recommendation_id}/accept - Accept recommendation
PUT /recommendations/{recommendation_id}/reject - Reject recommendation

Advisory

GET /advisory/{user_id} - Get personalized advisory

Claims Management

POST /claims - File new claim
GET /claims/{claim_id} - Get claim details
GET /claims/user/{user_id} - Get user claims
PUT /claims/{claim_id}/status - Update claim status

📊 Data Models

User Model

Field Type Description
id Integer Primary key
email String User email (unique)
name String Full name
age Integer User age
health_status String Good, Fair, or Poor

Insurance Policy Model

Field Type Description
id Integer Primary key
policy_name String Unique policy name
policy_type String Health, Life, Auto, or Property
coverage_amount Float Coverage in currency units
monthly_premium Float Monthly cost

Recommendation Model

Field Type Description
id Integer Primary key
user_id Integer Foreign key to User
policy_id Integer Foreign key to Policy
recommendation_score Float Score 0-100
is_accepted Boolean Recommendation status

Claim Model

Field Type Description
id Integer Primary key
user_id Integer Foreign key to User
policy_id Integer Foreign key to Policy
claim_amount Float Claimed amount
claim_status String Pending, Approved, or Rejected

🚀 Getting Started

Prerequisites

  • Python 3.8 or higher
  • pip package manager
  • Postman (for API testing)

Installation & Setup

  1. Navigate to project directory:
    cd "c:\Users\nites\Desktop\ai insurance"
  2. Install dependencies:
    pip install -r requirements.txt
  3. Start the FastAPI server:
    uvicorn main:app --reload
  4. Access the API:
    http://localhost:8000
  5. Access API documentation:
    http://localhost:8000/docs (Swagger UI)

📝 API Usage Examples

1. Create a User

curl -X POST "http://localhost:8000/users" \ -H "Content-Type: application/json" \ -d '{ "email": "john.doe@example.com", "name": "John Doe", "age": 35, "health_status": "Good" }'

2. Create an Insurance Policy

curl -X POST "http://localhost:8000/policies" \ -H "Content-Type: application/json" \ -d '{ "policy_name": "Premium Health Plan", "policy_type": "Health", "coverage_amount": 500000, "monthly_premium": 450, "description": "Comprehensive health insurance" }'

3. Create a Recommendation

curl -X POST "http://localhost:8000/recommendations" \ -H "Content-Type: application/json" \ -d '{ "user_id": 1, "policy_id": 1, "recommendation_score": 85.5, "reason": "Based on your profile, this plan is ideal" }'

4. Get Personalized Advisory

curl -X GET "http://localhost:8000/advisory/1"

5. File a Claim

curl -X POST "http://localhost:8000/claims" \ -H "Content-Type: application/json" \ -d '{ "user_id": 1, "policy_id": 1, "claim_amount": 50000, "description": "Hospital treatment" }'

🧪 Testing with Postman

Steps to Import Collection:

  1. Open Postman (download from postman.com if not installed)
  2. Click "Import" in the top left corner
  3. Select "Upload Files" and choose Postman_Collection.json
  4. Collection will be imported with all endpoints organized
  5. Ensure the server is running at http://localhost:8000
  6. Start testing by running requests from the collection

Suggested Testing Flow:

  1. Create 2-3 users using "Create User" requests
  2. Create 4-5 policies (different types) using "Create Policy" requests
  3. Create recommendations linking users to policies
  4. Get personalized advisory using "Get User Advisory"
  5. Accept/Reject recommendations to update status
  6. File claims and check claim status

💾 Database Information

SQLite Database

The application uses SQLite for persistent data storage. Database file: insurance.db

Tables Created:

  • users - User profiles and information
  • insurance_policies - Available insurance policies
  • recommendations - Policy recommendations for users
  • claims - Insurance claims filed by users

Database Auto-Creation:

Tables are automatically created when the FastAPI application starts for the first time. No manual database setup required!

🔧 Troubleshooting

Port Already in Use

If port 8000 is already in use, start the server on a different port:

uvicorn main:app --reload --port 8001

Database Errors

To reset the database, delete insurance.db and restart the application:

del insurance.db

Missing Dependencies

Ensure all packages are installed:

pip install -r requirements.txt --upgrade
